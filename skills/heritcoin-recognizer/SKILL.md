# Heritcoin Recognizer

用于识别、鉴定、估价单枚硬币。一次任务只处理同一枚钱币的 2 张图片，并且只调用一次识别脚本。

## 何时使用

- 用户提供聊天附件图片、图片 URL 或本地图片路径。
- 用户想知道这是什么币、年份、国家或地区、版别、铸记、材质、工艺、品相、价格等。
- 任务目标是识别“单枚钱币”，不是批量识别多枚钱币。

## 核心原则

- 一次识别任务 = 同一枚钱币的 2 张图片。
- 不要直接用模型原生视觉回答，必须走脚本。
- 不要把每张图单独识别，也不要把一次任务拆成多次调用。
- 只要已经拿到同一枚钱币的 2 张图，就直接识别；不要额外要求必须是“正反面”。
- 超过 2 张图时直接拒绝：`一次最多上传2张图，请重新上传`

## 最小工作流

1. 收集当前任务图片
   支持三类输入：

- 聊天附件图片
- 显式图片 URL
- 本地图片路径

2. 如果输入里有聊天附件，先解析附件路径
   在 `scripts/` 目录运行：

```bash
node resolve-chat-images.ts
```

把返回的 `images` 数组视为当前任务可执行图片路径。

3. 合并最近一个未完成任务

- 如果最近一个未完成任务只有 1 张图，且本条消息明确是在补同一枚钱币的第 2 张图，则并入该任务。
- 如果 assistant 上一条最终回复刚刚明确要求“还差一张”或“请补另一面/第二张”，那么紧接着下一条用户消息里的单张显式图片 URL / 本地路径，可以自动并入该任务。
- 除上述情况外，单张显式图片 URL / 本地路径默认视为新任务的第 1 张图，不能和历史未完成任务自动拼接。
- assistant 已经给出识别结果后，该任务视为完成，后续图片必须开启新任务。

4. 按当前任务图片数决定动作

- `0` 张：提示用户上传同一枚钱币的 2 张图片，并明确收到 2 张后即可继续识别和估价。
- `1` 张：明确当前只收到 1 张，还缺同一枚钱币的另一张图；同时明确用户补齐后即可继续识别和估价；不要调用识别。
- `2` 张：调用识别脚本 1 次。
- `>2` 张：直接回复 `一次最多上传2张图，请重新上传`，不要调用识别。

5. 只使用这个识别入口
   在 `scripts/` 目录运行：

```bash
node recognize.ts <img1> <img2> [token] [locale]
```

- 不要执行 `npm install`。
- 除非用户明确提供 `token`，否则使用脚本自身认证配置。
- 输出语言跟随用户当前请求语言；用户用中文提问时，最终结果必须输出中文。
- 当用户语言很明确时，优先显式传入对应 `locale`。

6. 最终输出契约

- 脚本成功时，最终回复必须原样返回脚本生成的 Markdown 最终识别结果；不要附加“上传中”“上传完成”等过程日志。
- agent 不得改写、压缩、摘要、重排，也不得省略任何非空字段。
- 脚本成功时，最终回复必须满足以下格式：
  - 顶部标题使用 `## 识别结果` 或对应 locale 标题。
  - 主体使用两列表格，列名固定为“项目 | 详情”或对应 locale 标签。
  - 行顺序上把“名称/币名”和“估价”放在最前面，其后依次输出年份、国家或地区、面值、铸造量及其它非空字段。
  - 收藏建议放在表格后，单独一行输出，不放进表格。
  - 不要使用 Markdown 代码块包裹最终结果。
  - 在结果结尾追加 1 行简短、保守的收藏建议。
  - 字段缺失时直接省略，不要臆造。
- 只有脚本返回错误信息，或用户明确要求另一种展示格式时，才允许整理；即便整理，也必须保留全部非空字段。

## 用户交互约束

- 用户一次发来 2 张图时，直接识别 1 次。
- 用户先发 1 张，后续补第 2 张时，在第 2 张到达后识别 1 次。
- 用户只发 1 张图时，不要只机械地说“还缺一张”；要明确告诉用户补齐同一枚钱币的另一张图后，就可以继续帮他识别和估价。
- assistant 明确催补图后，只有紧接着的下一条用户消息单独发 1 个显式图片 URL / 本地路径时，才自动把它并入当前未完成任务。
- 如果用户只是单独发 1 个显式图片 URL / 本地路径，且没有补图语义，也没有刚收到 assistant 的补图提示，默认把它当成新任务第 1 张图。
- 用户先连续发来 2 张聊天附件，随后再单独发一句“这是什么币？”时，直接用最近任务中的 2 张附件识别 1 次。
- 已经凑齐 2 张图后，不要再说“还差一张”。
- 不要在同一次任务中重复调用脚本。
- 用户上传超过 2 张图时，直接提示“`一次最多上传2张图，请重新上传`”，不要让用户再指定其中哪 2 张。
- 不要把当前消息中的多张图自动拆成多枚钱币，除非用户明确说是多枚钱币。
- 不要引用已完成旧任务里的图片或结果。
- 不要把历史未完成任务里的图片，和当前无补图语义的单张显式 URL / 本地路径自动拼成一次识别。
- 不要把脚本返回的完整结果改写成更短摘要，导致材质、直径、厚度、重量等非空字段丢失。
- 用户当前请求是中文时，不要输出英文结果；除非用户明确要求英文。
- 收藏建议保持 1 行、轻量、保守，不要扩展成投资建议或长篇判断。
- 默认按 Markdown 标题加两列表格输出；不要退回普通纯文本逐行字段，也不要使用代码块包裹表格。
- 不要直接根据模型看图结果回答“这是什么币”。

## 结果字段要求

- 至少保留：币名、估价、年份、国家或地区、面值。
- 如果脚本返回了 Krause 编号、材质、直径、厚度、重量、正面描述、背面描述或其它补充字段，也必须保留。
- 字段名不要附加“来源：脚本”之类说明，除非用户明确要求解释来源。
